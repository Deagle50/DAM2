Historia de .NET Framework
↓

  • El porqué de .Net Framework Microsoft quería plantar cara a Java y otros frameworks basados en PHP, y por eso empezó su desarrollo en la década de los 90, en un principio bajo el nombre de Next Generation Windows Services (NGWS).
  • .Net Framework 1.0 Tras varias betas desde finales del 2000, Microsoft lanzó .Net Framework en su versión 1.0 en febrero del 2002. Era compatible con Windows NT 4.0, 98, Me, 2000 y XP, y el IDE para su desarrollo era Visual Studio .Net. A mediados de julio del 2009 terminó el soporte para esta plataforma.
  • .Net Framework 1.1 Lanzada a principios del 2003 con Visual Studio .NET 2003, era la primera versión incluida por defecto en un SO.
  • .Net Framework 2.0-3.5 Lanzadas entre 2005 y 2007, y con soporte hasta 2011 (3.5 tiene hasta 2028), incluyeron soporte para nuevas apis, y en la última versión parte del código fue liberado
  • .Net Framework 4.0-4.8 Incluyeron nuevos lenguajes como C# y Visual Basic .Net, mejoras en rendimiento en el compilador JIT y mayor soporte a paneles táctiles, así como mejoras en su uso
  • .Net Core Una de sus diferencias con respecto a .Net Framework es que ahora es de código abierto en su totalidad. Soporta C#, F# y Visual Basic .net. Su última versión es la 3.0, y en un futuro saldrá la 5, saltando la 4 para evitar confusión con .Net Framework 4. A su vez acabará el soporte para .Net Framework.
